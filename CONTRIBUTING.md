# Contributing

Thanks for taking a look.

If you want to contribute, the most useful changes are the practical ones:

- better ingestion support for more IDEs
- safer sanitization workflows before publishing
- clearer wiki schemas for real-world use
- improvements to `refresh-memory`
- better product-family normalization and candidate-merge review workflows
- better examples of durable `entities`, `concepts`, and `queries`

## Good pull requests

Good contributions usually have one of these shapes:

- a concrete workflow improvement
- a simplification that makes the repo easier to adopt
- a bug fix in the tooling
- a documentation fix based on real usage

## Before you open a PR

Please keep the starter clean:

- do not add your private chat history
- do not commit secrets, keys, or tokens
- do not publish company-internal wiki data as an example

If you want to show a pattern, use obviously fake placeholder content.

## Discussion

If you are not sure whether an idea belongs in the starter, open an issue first. Friendly suggestions are welcome.
